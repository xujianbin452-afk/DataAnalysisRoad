1. 什么是信贷业务；

4. 风险数据分析岗和风控策略岗有什么区别；

# 2. 什么是信贷业务？

信贷业务本质上是：

> 金融机构把钱借给用户，用户按约定时间还本金和利息。

常见信贷产品包括：

* 信用卡；
* 消费贷；
* 现金贷；
* 分期贷；
* 车贷；
* 房贷；
* 小微企业贷款；
* 经营贷。

信贷业务的核心矛盾是：

> 既要把钱借出去赚利息，又要控制借出去的钱能不能收回来。

所以信贷业务不是简单追求“放款越多越好”，而是在以下两者之间做平衡：

| 目标   | 含义                |
| ---- | ----------------- |
| 业务规模 | 放款金额、通过率、用户数、收入   |
| 风险控制 | 逾期率、坏账率、风险成本、资产质量 |

风控的价值就在于：

> 在尽量不误伤好客户的前提下，识别和控制坏客户风险。

---

# 3. 什么是好客户和坏客户？

在信贷风控里，用户通常会被分为：

## 3.1 好客户

好客户通常指：

* 按时还款；
* 没有严重逾期；
* 信用表现稳定；
* 风险较低；
* 可能带来稳定收益。

## 3.2 坏客户

坏客户通常指：

* 发生严重逾期；
* 长时间不还款；
* 未来违约概率较高；
* 可能造成坏账损失。

在项目里，我们通常需要人为定义坏客户标签，例如：

```text
bad_flag = 1 表示坏客户
bad_flag = 0 表示好客户
```

在 Give Me Some Credit 数据集中，可以使用目标变量：

```text
SeriousDlqin2yrs
```

作为坏客户标签。

常见理解：

```text
SeriousDlqin2yrs = 1：未来两年发生严重逾期 / 财务困境
SeriousDlqin2yrs = 0：未来两年未发生严重逾期
```

# 4. 信贷风控三阶段：贷前、贷中、贷后

信贷风控不是只在放款前发生，而是贯穿整个贷款生命周期。

---

## 4.1 贷前风控

贷前风控发生在用户申请贷款时。

核心问题：

> 这个用户能不能借？能借多少？利率多少？

贷前风控主要包括：

| 环节   | 说明                 |
| ---- | ------------------ |
| 身份核验 | 判断用户身份是否真实         |
| 反欺诈  | 判断是否存在黑产、虚假申请、冒用身份 |
| 准入策略 | 判断用户是否满足基本申请条件     |
| 信用评估 | 评估用户未来违约风险         |
| 授信额度 | 决定给用户多少额度          |
| 定价策略 | 决定贷款利率或费用          |
| 审批决策 | 通过、拒绝、人工审核         |

贷前常见策略：

```text
年龄过低或过高 → 拒绝或人工审核
历史严重逾期次数过多 → 拒绝
收入过低且负债较高 → 降额或拒绝
信用评分较高 → 正常通过
信用评分较低 → 拒绝或低额度
```

贷前核心指标：

* 申请人数；
* 审批通过人数；
* 通过率；
* 拒绝率；
* 放款金额；
* 首逾率；
* 坏账率；
* 风险收益。

---

## 4.2 贷中风控

贷中风控发生在用户获得贷款或额度之后。

核心问题：

> 用户风险有没有变化？是否需要预警、降额、冻结？

贷中风控主要包括：

| 环节   | 说明                 |
| ---- | ------------------ |
| 行为监控 | 观察用户使用额度、借款、还款行为   |
| 风险预警 | 识别风险变高的用户          |
| 额度调整 | 对风险升高用户降额，对优质用户提额  |
| 账户管理 | 冻结、限制、提醒           |
| 异常识别 | 识别异常交易、异常借款、异常还款行为 |

贷中常见策略：

```text
用户短期内频繁借款 → 预警
额度使用率长期过高 → 降额
出现轻微逾期 → 加强提醒
风险评分恶化 → 冻结部分额度
```

贷中核心指标：

* 额度使用率；
* 借款频次；
* 还款行为；
* 风险评分变化；
* 贷中预警命中率；
* 降额后逾期率变化。

---

## 4.3 贷后风控

贷后风控发生在用户逾期之后。

核心问题：

> 用户逾期后，如何提高回收、降低损失？

贷后风控主要包括：

| 环节     | 说明                 |
| ------ | ------------------ |
| 逾期提醒   | 短信、电话、App提醒        |
| 催收分层   | 根据逾期天数和风险等级分配催收策略  |
| 还款意愿识别 | 判断用户是不想还、不能还，还是忘记还 |
| 回收策略   | 制定不同催收强度           |
| 坏账核销   | 长期无法回收时进行坏账处理      |

贷后常见策略：

```text
M0轻微逾期 → 自动短信提醒
M1逾期 → 电话催收
M2/M3逾期 → 高强度催收或委外处理
高风险大额用户 → 优先催收
低风险小额用户 → 温和提醒
```

贷后核心指标：

* 逾期率；
* M0/M1/M2/M3占比；
* 回收率；
* 催回率；
* 迁徙率；
* 坏账率；
* 核销率。

# 5. 风险数据分析岗和风控策略岗的区别

## 5.1 风险数据分析岗

风险数据分析岗更偏“看清风险”。

核心工作：

1. 提取信贷数据；
2. 搭建风险指标；
3. 监控资产质量；
4. 分析逾期和坏账原因；
5. 输出风险分析报告；
6. 支持策略同学做决策。

典型工作内容：

```text
分析不同渠道坏账率
监控每日申请量和通过率
分析不同客群逾期率
定位某个月坏账率升高原因
搭建风险看板
输出贷后资产质量周报
```

需要能力：

* SQL；
* Python/Pandas；
* Excel；
* Tableau/BI；
* 风控指标理解；
* 业务分析能力；
* 报告表达能力。

---

## 5.2 风控策略岗

风控策略岗更偏“制定规则”。

核心工作：

1. 制定准入规则；
2. 调整审批策略；
3. 设计额度策略；
4. 制定贷后催收策略；
5. 评估策略上线效果；
6. 平衡通过率、坏账率和收益。

典型工作内容：

```text
是否拒绝某类高风险用户
是否降低某些用户的授信额度
新策略上线后通过率下降多少
坏账率是否改善
是否误伤好客户
是否需要灰度实验
```

需要能力：

* 风控业务理解；
* SQL/Python；
* 数据分析；
* 策略推演；
* A/B测试；
* 归因分析；
* 模型分数理解；
* 跨部门沟通。

---

# 6. 风控策略的本质

风控策略不是简单地“拒绝风险高的人”。

如果全部拒绝高风险用户，坏账率可能下降，但业务规模也会下降。

所以风控策略的本质是：

> 在风险和收益之间做平衡。

需要同时关注：

| 指标   | 含义        |
| ---- | --------- |
| 通过率  | 业务规模      |
| 拒绝率  | 风控严格程度    |
| 坏账率  | 资产质量      |
| 放款金额 | 业务收入基础    |
| 风险成本 | 坏账损失      |
| 收益   | 利息、手续费等收入 |
| 误伤率  | 被拒绝的好客户比例 |

好的风控策略应该是：

```text
拒绝真正高风险用户
减少对低风险用户的误伤
让业务规模和资产质量同时可控
```

# 7. 第一个信贷逾期风险分析项目要解决什么问题？

本项目目标：

> 基于用户历史信用、收入、负债、年龄等信息，分析哪些用户更容易发生严重逾期，并提出初步风控策略建议。

项目核心问题：

1. 整体坏客户比例是多少？
2. 哪些变量和坏账率关系更明显？
3. 哪些客群风险最高？
4. 是否可以根据规则做低/中/高风险分层？
5. 拒绝高风险用户后，通过率和坏账率会如何变化？
6. 可以提出哪些准入、额度、贷后策略？

# Day 2：核心信贷指标与风险分析口径

## 1. 今天学习目标

今天目标是掌握风控分析中最常用的指标。

必须掌握：

1. 申请量；
2. 通过率；
3. 拒绝率；
4. 放款量；
5. 逾期率；
6. 坏账率；
7. DPD；
8. M0/M1/M2/M3；
9. 首逾率；
10. 风险成本；
11. 回收率；
12. 核销率。

这些指标是风险数据分析和风控策略岗位的基础语言。

---

# 2. 申请量

## 2.1 定义

申请量指在某个时间周期内提交贷款申请的用户数量或申请单数量。

常见口径：

```text
申请量 = 提交贷款申请的订单数
申请人数 = 提交贷款申请的去重用户数
```

## 2.2 业务含义

申请量反映业务流量规模。

申请量上升可能来自：

* 投放增加；
* 渠道扩张；
* 活动促销；
* 产品利率下降；
* 用户需求上升。

申请量下降可能来自：

* 流量减少；
* 风控变严；
* 产品竞争力下降；
* 审批体验差；
* 市场环境变化。

## 2.3 分析时要注意

申请量不能单独看，要结合：

* 通过率；
* 放款率；
* 坏账率；
* 渠道结构；
* 客群结构。

如果申请量上升，但坏账率也上升，可能说明新增流量质量较差。

---

# 3. 通过率

## 3.1 定义

通过率表示申请用户中有多少被审批通过。

```text
通过率 = 审批通过人数 / 申请人数
```

或：

```text
通过率 = 审批通过订单数 / 申请订单数
```

## 3.2 业务含义

通过率越高，说明策略越宽松，业务规模可能更大。

但通过率过高可能带来：

* 风险客群进入；
* 逾期率上升；
* 坏账率上升；
* 风险成本升高。

通过率过低可能带来：

* 业务规模下降；
* 收入下降；
* 误伤好客户；
* 用户体验变差。

## 3.3 分析方式

常见分析维度：

```text
按渠道看通过率
按城市看通过率
按年龄段看通过率
按信用等级看通过率
按产品类型看通过率
按时间趋势看通过率
```

---

# 4. 拒绝率

## 4.1 定义

```text
拒绝率 = 审批拒绝人数 / 申请人数
```

由于：

```text
通过率 + 拒绝率 ≈ 100%
```

所以拒绝率可以反映风控策略严格程度。

## 4.2 业务含义

拒绝率高，不一定是坏事。

可能说明：

* 风控严格；
* 风险拦截能力强；
* 申请客群质量差；
* 某渠道流量风险高。

但拒绝率太高也可能说明：

* 策略过严；
* 误伤好客户；
* 用户体验差；
* 业务规模受损。

---

# 5. 放款量与放款金额

## 5.1 定义

放款量：

```text
放款量 = 实际成功放款订单数
```

放款金额：

```text
放款金额 = 实际发放贷款金额总和
```

## 5.2 业务含义

放款量和放款金额代表业务规模。

常见分析：

```text
按月放款金额趋势
按渠道放款金额
按用户风险等级放款金额
按额度区间放款金额
```

如果高风险用户贡献了大量放款金额，要重点关注资产质量。

---

# 6. 逾期率

## 6.1 定义

逾期率表示发生逾期的客户占比。

```text
逾期率 = 逾期客户数 / 放款客户数
```

也可以按金额计算：

```text
金额逾期率 = 逾期贷款余额 / 总贷款余额
```

## 6.2 常见口径

```text
DPD1+逾期率：逾期1天及以上客户占比
DPD7+逾期率：逾期7天及以上客户占比
DPD30+逾期率：逾期30天及以上客户占比
DPD90+逾期率：逾期90天及以上客户占比
```

## 6.3 业务含义

逾期率越高，说明资产质量越差。

但分析时必须明确口径：

```text
是客户数逾期率，还是金额逾期率？
是 DPD1+，还是 DPD30+？
分母是放款客户，还是到期客户？
```

不同口径不能混用。

---

# 7. 坏账率

## 7.1 定义

坏账率通常表示最终无法收回或严重逾期的贷款占比。

```text
坏账率 = 坏账客户数 / 放款客户数
```

或：

```text
金额坏账率 = 坏账金额 / 放款金额
```

## 7.2 坏账和逾期的区别

逾期不一定是坏账。

例如：

```text
逾期3天：可能只是忘记还款
逾期30天：风险明显增加
逾期90天以上：通常更接近坏账定义
```

常见做法：

```text
DPD30+ 可以作为风险客户
DPD90+ 可以作为严重坏账客户
```

具体口径要看公司业务规定。

## 7.3 项目中如何定义坏客户

在 Give Me Some Credit 项目中：

```text
SeriousDlqin2yrs = 1
```

可以作为坏客户标签。

即：

```text
bad_flag = SeriousDlqin2yrs
```

---

# 8. DPD

## 8.1 定义

DPD 是 Days Past Due 的缩写，表示逾期天数。

```text
DPD = 当前日期 - 应还款日期
```

如果用户应在 6 月 1 日还款，但 6 月 10 日仍未还款：

```text
DPD = 9
```

## 8.2 常见口径

| 口径     | 含义       |
| ------ | -------- |
| DPD1+  | 逾期1天及以上  |
| DPD7+  | 逾期7天及以上  |
| DPD30+ | 逾期30天及以上 |
| DPD60+ | 逾期60天及以上 |
| DPD90+ | 逾期90天及以上 |

## 8.3 业务含义

DPD越高，风险越严重。

一般来说：

```text
DPD1+：轻微逾期
DPD30+：明显风险
DPD90+：严重风险
```

---

# 9. M0 / M1 / M2 / M3

## 9.1 定义

M 分层表示逾期阶段。

| 阶段 | 通常含义     |
| -- | -------- |
| M0 | 逾期0-30天  |
| M1 | 逾期31-60天 |
| M2 | 逾期61-90天 |
| M3 | 逾期90天以上  |

不同公司口径可能略有不同，但大方向一致。

## 9.2 业务含义

M0：

```text
轻度逾期，可能是忘记还款，催回概率较高。
```

M1：

```text
风险明显上升，需要更积极催收。
```

M2：

```text
回收难度增加，进入高风险阶段。
```

M3：

```text
严重逾期，可能接近坏账。
```

## 9.3 贷后策略

| 阶段 | 常见策略            |
| -- | --------------- |
| M0 | 短信提醒、App提醒、自动外呼 |
| M1 | 电话催收、重点跟进       |
| M2 | 强催收、分案处理        |
| M3 | 委外催收、法律手段、核销准备  |

---

# 10. 首逾率

## 10.1 定义

首逾率通常指用户第一次还款就发生逾期的比例。

```text
首逾率 = 首期发生逾期客户数 / 首期应还客户数
```

## 10.2 业务含义

首逾率是贷前风控非常关注的指标。

如果首逾率高，可能说明：

* 贷前准入过松；
* 欺诈风险高；
* 用户本身还款意愿差；
* 渠道质量差；
* 风控模型识别不足。

首逾率高通常比后期逾期更危险，因为用户刚借款就逾期，说明前端风控可能存在明显问题。

---

# 11. 回收率

## 11.1 定义

回收率表示逾期金额中有多少被追回。

```text
回收率 = 已回收逾期金额 / 逾期金额
```

## 11.2 业务含义

回收率衡量贷后催收效果。

回收率高说明：

* 催收策略有效；
* 客户还款意愿较强；
* 逾期客户仍有偿还能力。

回收率低说明：

* 逾期客户质量差；
* 催收策略弱；
* 坏账风险高。

---

# 12. 核销率

## 12.1 定义

核销是指贷款长期无法收回后，从账面上确认损失。

```text
核销率 = 核销金额 / 放款金额
```

## 12.2 业务含义

核销率越高，说明最终损失越严重。

核销率一般是滞后指标，不会立刻体现风险，而是在贷款逾期较长时间后体现。

---

# 13. 风险成本

## 13.1 定义

风险成本可以理解为坏账损失占业务规模的比例。

```text
风险成本 = 坏账损失 / 放款金额
```

简化理解：

```text
风险成本越高，说明赚的钱被坏账吃掉得越多。
```

## 13.2 业务含义

风控不是只看坏账率，还要看风险成本和收益。

例如：

| 策略  | 通过率 | 坏账率 |   收益 |
| --- | --: | --: | ---: |
| 策略A |   高 |   高 | 不一定高 |
| 策略B |   中 |   中 | 可能更优 |
| 策略C |   低 |   低 | 规模太小 |

最优策略不一定是坏账率最低，而是风险收益最优。

---

# 14. 指标之间的关系

## 14.1 通过率和坏账率

通常情况下：

```text
通过率提高 → 更多风险用户进入 → 坏账率可能升高
通过率降低 → 风控更严格 → 坏账率可能下降
```

但如果模型和策略能力强，也可能做到：

```text
通过率稳定，坏账率下降
通过率提高，坏账率不明显上升
```

这就是风控策略优化的价值。

## 14.2 申请量和坏账率

申请量上升不一定是好事。

需要看新增申请来自哪里：

```text
优质渠道增加 → 申请量上升且坏账率稳定
劣质渠道增加 → 申请量上升但坏账率恶化
```

## 14.3 拒绝率和误伤

拒绝率高不代表策略一定好。

如果拒绝了大量本来会正常还款的好客户，就叫误伤。

风控策略要避免：

```text
过度保守
只追求低坏账率
牺牲全部业务规模
```

# 2. 什么是客群风险分析？

客群风险分析就是：

> 把用户按照某些特征分组，然后比较不同组的坏账率差异。

例如：

按年龄分组：

| 年龄段   |  用户数 | 坏客户数 | 坏账率 |
| ----- | ---: | ---: | --: |
| 18-24 | 1000 |  120 | 12% |
| 25-34 | 5000 |  350 |  7% |
| 35-44 | 6000 |  300 |  5% |
| 45-54 | 4000 |  160 |  4% |
| 55+   | 3000 |   90 |  3% |

如果 18-24 岁坏账率明显更高，就说明年龄可能具有一定风险区分能力。

---

# 3. 为什么风控要看分组坏账率？

因为风控策略通常不是直接看单个用户，而是先从群体规律中识别风险特征。

例如：

```text
低收入用户坏账率更高
额度使用率高的用户坏账率更高
历史逾期次数多的用户坏账率更高
年轻用户坏账率更高
负债率高的用户坏账率更高
```

这些发现可以转化为策略：

```text
低收入 + 高负债用户 → 降额或人工审核
历史严重逾期用户 → 拒绝或强限制
额度使用率极高用户 → 加强贷中监控
低风险稳定用户 → 提额或优先通过
```

## 4.2 第二步：分组

### 类别变量

类别变量可以直接分组，例如：

```text
性别
学历
城市
渠道
婚姻状态
```

### 连续变量

连续变量需要分箱，例如：

```text
年龄
收入
负债率
额度使用率
贷款金额
```

常见分箱方法：

| 方法    | 说明        |
| ----- | --------- |
| 等宽分箱  | 每组区间宽度相同  |
| 等频分箱  | 每组样本数量接近  |
| 业务分箱  | 按业务经验设定区间 |
| 决策树分箱 | 用模型寻找切分点  |

第一个项目优先使用：

```text
业务分箱 + 等频分箱
```

---

## 4.3 第三步：计算每组坏账率

每组需要计算：

```text
total_cnt：用户数量
bad_cnt：坏客户数量
bad_rate：坏账率
```

公式：

```text
bad_rate = bad_cnt / total_cnt
```

Pandas 通用写法：

```python
result = (
    df.groupby('group_col')
      .agg(
          total_cnt=('bad_flag', 'count'),
          bad_cnt=('bad_flag', 'sum'),
          bad_rate=('bad_flag', 'mean')
      )
      .reset_index()
)
```

## 4.4 第四步：判断趋势

分析一个变量是否有用，要看：

1. 不同组坏账率是否差异明显；
2. 是否存在单调趋势；
3. 高风险组样本量是否足够；
4. 是否符合业务常识；
5. 是否可以转化为策略。

例如：

```text
额度使用率越高，坏账率越高
```

这就是比较好的风险变量。

但如果某变量分组结果是：

```text
第一组 5%
第二组 12%
第三组 6%
第四组 15%
第五组 7%
```

波动很乱，就要谨慎判断。

# 5. 如何判断一个变量是否有风控价值？

一个变量有风控价值，通常需要满足：

## 5.1 区分度明显

不同分组坏账率差异明显。

例如：

```text
低额度使用率组坏账率 3%
高额度使用率组坏账率 15%
```

说明该变量能区分风险。

## 5.2 趋势合理

变量变化和风险变化符合业务逻辑。

例如：

```text
历史逾期次数越多，坏账率越高
```

这是合理趋势。

## 5.3 样本量足够

不能只看坏账率，还要看样本量。

例如：

```text
某组坏账率 50%，但只有 2 个用户
```

这种结果不稳定，不能直接制定策略。

## 5.4 可以策略化

变量结果要能转化为策略。

例如：

```text
历史90天逾期次数 > 0 的用户坏账率明显高
```

可以转化为：

```text
准入拒绝 / 人工审核 / 降额
```

# 6. 风险分层是什么？

风险分层是把用户按照风险高低分为几类。

常见分层：

```text
低风险
中风险
高风险
```

也可以更细：

```text
A/B/C/D/E
```

风险分层的作用：

1. 支持准入策略；
2. 支持额度策略；
3. 支持定价策略；
4. 支持贷后监控；
5. 支持差异化运营。

例如：

| 风险等级 | 策略          |
| ---- | ----------- |
| 低风险  | 正常通过，可适度提额  |
| 中风险  | 正常通过但降低额度   |
| 高风险  | 拒绝、人工审核或强监控 |
